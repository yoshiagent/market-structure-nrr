市場結構 + N-RR 台股交易系統

框架設計文件 v1.0 ｜ 為 ETF 化、法人化、趨勢延長的台股環境設計的多層次交易系統

1. 策略哲學

核心信念
先判斷市場環境，再確認個股趨勢成立，最後用 RR 決定資金配置到哪一檔相對強勢股。
目標不是抄到最低點，而是趨勢成立後的資金效率最大化。

與傳統配對交易的差異

面向傳統 Pair Trading市場結構 + N-RR
核心假設價差會均值回歸強勢產業內的資金會輪動
持有方向市場中性 (多空各一)同向多單為主、強弱輪換
觸發訊號Ratio Z-score 超過門檻立即進場極值 + 折返確認 + 分批換股
市場環境通常忽略大盤方向環境不對 (外資主導空頭) 直接停做
賺取的錢價差收斂趨勢報酬 + 輪動超額

為什麼這套架構適合現在的台股

  • ETF 化：高股息 ETF + 市值型 ETF 受益人破千萬，內資成為長線買盤
  • 法人化：外資期現貨部位 + 投信買超主導日內方向
  • 趨勢延長：AI、HPC、先進封裝題材長線，月線級別波段比短線 mean reversion 容易做
  • 強勢產業內輪動：同題材的股票會「你漲一波我漲一波」，這就是 RR 賺的錢

2. 三層架構總覽

Layer 1 ｜ Market Structure

市場結構

判斷今天市場由誰主導：外資 / ETF 投信 / 壽險 / 匯率流動性。產出 MarketStructureScore，決定能不能交易。

Layer 2 ｜ N-Shape Trend

N 形波段

確認個股月/週/日線結構是否進入 N 形上升趨勢。判斷該不該進場。

Layer 3 ｜ Relative Rotation

RR 相對輪動

在同一強勢產業中比較兩檔股票的相對強弱，決定資金放哪一檔。

三層的順序不可顛倒。
若 Layer 1 = 外資主導空頭，Layer 2/3 全部停做，只留現金或避險部位。

4. 市場結構分數與判讀

4.1 ForeignPressureScore（外資壓力分數）

ForeignPressureScore = 0.30 × Z_ForeignSpot_60 // 外資現貨買賣超 Z (買超為正) + 0.30 × (-Z_ForeignFut_60) // 外資期貨淨空 Z (淨空為負，取負號) + 0.20 × (-Z_TWD_60) // 台幣升貶 Z (升值為負，取負號 → 升值為正) + 0.20 × Z_SOX_or_TSM_ADR_Ret_60 // SOX/TSM ADR 報酬 Z

4.2 ETFSupportScore（內資承接分數）

ETFSupportScore = 0.35 × Z_Trust_60 // 投信買超 Z + 0.25 × Z_ETF_HolderMoM_60 // ETF 受益人月增 Z + 0.20 × Z_HiDiv_Volume_Pct_60 // 高股息 ETF 成交比重 Z + 0.20 × Z_MktDrawdownNarrow_60 // 大盤跌幅收斂程度 Z
大盤跌幅收斂程度 的算法：當大盤拉回時，若跌幅遞減 (今日跌 -0.3% 比昨日 -1.2% 收斂) → 給正分。 公式範例：MktDrawdownNarrow = -ΔTAIEX_Return，再做 60 日 Z-score。

4.3 MarketStructureScore（市場結構總分）

MarketStructureScore = ETFSupportScore − ForeignPressureScore

4.4 判讀對照表

分數區間環境判讀策略動作建議倉位
> +1.0ETF/內資主導，慢牛環境積極做多、RR 全開80~100%
+0.3 ~ +1.0承接有效，偏多分批做多、RR 開啟50~80%
−0.3 ~ +0.3多空拉鋸只做 N 形最強股、RR 縮量30~50%
−1.0 ~ −0.3外資壓力偏大停止新進、舊倉收斂10~30%
< −1.0外資主導空頭RR 全停、出清非核心0~10%

5. 第二層：N 形波段模組

N 形定義：第一波上漲 → 回檔不破關鍵支撐 → 再突破前高 → 形成「N」字形態。
N 形成立才代表趨勢真的延續，避免只看單日突破就追進。

5.1 月線層 (Macro Trend)

  • 月 K 脫離盤整區：過去 6~12 個月的箱型上緣被突破
  • 月線低檔轉強：MA12 (12 月 MA) 由下彎轉為持平 / 向上
  • 月線量能放大：突破月 K 的量 ≥ 過去 12 個月平均量 × 1.5
  • 月線不破前低：回檔最低點高於前一次明顯低點

5.2 週線層 (Trend Confirmation)

  • 週 MA20 向上：週線級別 20 週均線斜率為正
  • 股價回測週 MA20 不破，或跌破後重新站回
  • 週 K 高低點墊高：HH (Higher High) + HL (Higher Low) 連續 2 組以上
  • 週量價結構健康：上漲週量 > 下跌週量

5.3 日線層 (Entry Trigger)

  • 日線回測 MA20 / MA60：回到均線附近，未跌破
  • 突破前高：以收盤價突破過去 N 日高點
  • 回測不破：突破後若拉回，低點不破突破點
  • 量縮整理、量增突破：整理時量縮、突破時量增至 5 日均量 × 1.5 以上

5.4 N 形狀態判定

狀態判定條件動作
未成立月/週線結構未滿足觀察，不動作
第一波形成中突破盤整但未回檔觀察，等回檔
回測中回測週 MA20 或前高位置準備分批進場
N 形成立回測不破 + 再突破前高進場主力部位
N 形失敗跌破回測低點 + 跌破週 MA20出場停損

6. 第三層：RR 模組

RR = Relative Rotation，不是傳統 Pair Trading。
目的：在同一強勢產業中，把資金配置到相對更有優勢的股票，提高超額報酬。

6.2 RR 指標公式

// 兩種計算方式擇一 Ratio = Stock_A_Close / Stock_B_Close Spread = Stock_A_Return − Stock_B_Return // 衍生指標 Ratio_MA20 = MA(Ratio, 20) Ratio_MA60 = MA(Ratio, 60) Ratio_Z60 = (Ratio − Ratio_MA60) / Ratio_Std60 Ratio_MA20_Slope = Ratio_MA20.diff(5) Ratio_MA60_Slope = Ratio_MA60.diff(10) RS_Break = Ratio > max(Ratio[-60:-1]) RS_Pullback = Ratio < Ratio_MA20 & Ratio_Z60 > 0

6.3 RR 狀態

狀態判定意義
A 強Z60 > 0 且 Ratio 向上資金集中 A
B 強Z60 < 0 且 Ratio 向下資金集中 B
極值 (A 過強)Z60 > +2注意換股訊號

7. RR 換股邏輯（分批）

不要碰到 Z-score 門檻就立刻換股。
Z-score 是「相對強弱已過頭」的訊號，但「過頭」可以一直過頭。
正確流程：極值 → 折返確認 → 分批換股。

7.1 換股階段（以 A 過強為例）

階段觸發條件動作說明
1. 警示Z60 > +2不動A 過強，但趨勢可能繼續
2. 第一批Z60 > +2 維持 + Ratio 跌破 MA5A → B 換 30%短均線轉弱，折返開始
3. 第二批Ratio 跌破 MA20A → B 換 40%中期均線轉弱確認
4. 第三批Ratio 跌破 MA60 或 Z60 回到 0 以下A → B 換剩餘 30%輪動完成

7.2 反向 (B 過強) 邏輯對稱

  • Z60 < −2 → 警示
  • Ratio 站回 MA5 → B → A 換 30%
  • Ratio 站回 MA20 → B → A 換 40%
  • Ratio 站回 MA60 或 Z60 回到 0 以上 → 換剩餘 30%

7.3 為什麼分批

  • 避免「過頭可以更過頭」：強勢股可能繼續強 1~2 週
  • 避免「折返是假動作」：第一批先試水溫
  • 降低換股成本：分散在多日進出，吃到平均成交價

8. RR 必要濾網

所有濾網必須同時通過，RR 才能執行。
缺一不可。寧可不做，不可逆勢。
  1. 兩檔股票週 MA20 皆向上 → 確認都在多頭結構
  2. 兩檔股票皆未跌破主要 N 形結構 → 都還在趨勢內
  3. 所屬產業仍為強勢產業 → 用產業指數或 ETF 確認
  4. 市場結構不是外資主導空頭 → MarketStructureScore > −1.0
  5. Ratio 出現極值後有折返確認 → 不是 Z 一過 ±2 就換

8.1 應該避免的錯誤

常見錯誤為什麼錯正確做法
逆勢抄弱股弱者恆弱，趨勢未終結前抄底會持續虧等弱股自己出現 N 形再進
太早賣強股強勢股可能再強 30~50%等 Z60 折返 + Ratio 跌破 MA20
把趨勢誤判成均值回歸RR 不是 mean reversion，是輪動濾網沒過就不做
產業已轉弱還在輪動整個 sector 都跌，輪動沒意義產業指數跌破週 MA20 → 停做 RR

9. 動態 Z-score 門檻

Z-score 門檻不固定。市場越強，極值越極端；震盪盤門檻要放寬。

9.1 門檻表

環境判定條件Z-score 門檻
強多頭 / ETF 慢牛MarketStructureScore > +1.0 且 大盤站上週 MA20±2.5 ~ ±3.0
一般盤MarketStructureScore −0.3 ~ +1.0±2.0
空頭震盪MarketStructureScore < −0.3±1.5

9.2 門檻調整邏輯

def dynamic_z_threshold(market_score, mkt_above_wma20, sector_strong, ratio_slope): base = 2.0 if market_score > 1.0 and mkt_above_wma20: base = 2.5 if sector_strong and ratio_slope > 0: base = 3.0 # 強到不換 elif market_score < -0.3: base = 1.5 return base
為什麼強多頭門檻要拉高？
慢牛環境下強勢股可以一路強上去，過早觸發換股會把最賺的部分讓出去。

11. 回測框架

Look-ahead Bias 必須避免。
所有指標 t 日的訊號，只能使用 t-1 日（含）以前的資料。

11.1 回測項目

  1. 單純 N 形策略：忽略市場結構與 RR，只看 N 形進出
  2. 單純 RR 策略：忽略市場結構，只看 Ratio Z60 + 分批換股
  3. 市場結構 + N 形：加上市場結構過濾
  4. 市場結構 + N + RR (完整版)：三層全開

16. 警示條件

16.1 市場層警示

事件觸發建議
市場結構翻轉MarketStructureScore 由 > +0.3 跌破至 < −0.3檢視所有部位，準備降倉
外資壓力急升ForeignPressureScore 5 日內上升 > 1.5留意是否轉空
ETF 承接失效ETFSupportScore 連續 3 日 < 0內資撐盤動能轉弱
台幣急貶USDTWD 單日升幅 > +0.5%外資匯出風險

16.2 RR 層警示

事件觸發建議
RR 達極值|Z60| 超過動態門檻準備換股
RR 折返確認Z 超門檻後，Ratio 跌破 MA5執行第一批換股 (30%)
濾網失效5 個濾網有 1 個失效暫停該組 RR
N 形破壞跌破回測低點 + 跌破週 MA20停損出場

17. 常見錯誤與防呆機制

17.1 資料層

錯誤影響防呆
yfinance 抓不到當日資料訊號計算空缺retry 3 次 + fallback 到前一日
三大法人未公布市場結構分數錯誤檢查資料日期 ≥ 今日，否則中止寫入
除權息日未調整Ratio 跳動誤判用 yfinance 的 auto_adjust=True
停牌股票Z-score 計算扭曲檢查連續成交日，少於 N 日剔除

17.2 邏輯層

錯誤影響防呆
Look-ahead Bias回測虛胖所有指標只用 .loc[:date-1]
用收盤價當訊號又用收盤價成交當日無法執行訊號用 t-1，成交用 t 開盤
Z-score 窗太短過度敏感固定 60 日，且 require 至少 60 筆資料才出訊號
濾網忘記檢查逆勢開倉assert all(filters) 才允許 action

17.3 執行層

錯誤防呆
同一日多次執行造成資料重複寫入前檢查日期是否已存在
Excel 被使用者打開無法寫入try/except + 寫入暫存檔再 rename
排程跑失敗無人察覺失敗時寫 log + 觸發 LINE 警示
主機關機沒跑到下次開機檢查上次成功日期，補跑

18. AI Agent 擴充模組（未來）

目前框架是規則引擎 (rule-based)。所有決策可解釋、可回測、可信任。
AI Agent 擴充是輔助層，不取代規則，只補充規則無法處理的非結構化資訊。

18.1 可擴充方向

模組用途實作
新聞情緒分析抓取個股新聞，判斷正/負面Claude API + RSS / 鉅亨網爬蟲
法說會摘要解析法說會逐字稿，抽取財務指引Claude API + PDF 解析
產業趨勢追蹤追蹤海外重要報告 (SemiAnalysis 等)Web search + Claude 摘要
異常檢測找出 Z60 異常但規則沒抓到的情境Isolation Forest / LLM 解釋
策略對話介面用自然語言詢問「今天為什麼建議降倉？」Claude API + 訊號 JSON 為 context
每週週報自動生成用本週訊號 + 走勢自動寫一份報告Claude API + Jinja 模板 → HTML

18.2 設計原則

  • AI 不下決策：AI 只提供 context 與摘要，最終決策仍由規則引擎產出
  • AI 輸出要可審計：每次 AI 呼叫的輸入輸出存檔，可回溯
  • AI 失效時系統仍能運作：規則引擎獨立運作，AI 只是加值

19. 開發路線圖

Phase 1 ｜ 資料管道 (1~2 週)

  • 建 data/ 目錄結構
  • 寫好 5 個 fetcher (price / 3insti / etf / macro / futures)
  • 驗證資料完整性與更新時間
  • 建 config/pairs.yaml 與 params.yaml

Phase 2 ｜ 指標模組 (1~2 週)

  • market_structure.py：分數計算 + 環境判讀
  • n_shape.py：月/週/日 N 形狀態
  • rr.py：Ratio / Z60 / MA / 狀態
  • 單元測試每個指標的邊界情況

Phase 3 ｜ 決策引擎與回測 (2~3 週)

  • decision.py：整合三層輸出 action
  • backtest.py：4 種策略對照回測
  • 產出回測報告 (HTML)
  • 調參：動態門檻、分批比例

Phase 4 ｜ Dashboard 與自動化 (1~2 週)

  • dashboard.py：每日 HTML 儀表板
  • alerts.py：警示與 LINE Notify
  • Windows 排程器 + log 機制
  • GitHub Pages 部署

Phase 5 ｜ AI Agent 加值 (可選，未來)

  • 新聞情緒分析模組
  • 每週週報自動生成
  • 對話式策略諮詢
Phase 1 → Phase 4 為核心 MVP，預估 5~8 週。
完成後就有一套可每日自動跑、可回測、可推播的完整系統。
Phase 5 為 AI 加值，視需求加入。
